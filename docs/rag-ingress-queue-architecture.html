System architecture review

rag-ingress-queue

Local PC와 Mac mini에서 발생하는 redacted RAG-ready transcript, session, document ingest 요청을 단일 write path로 받아 downstream RAG target이 감당 가능한 속도로 전달하는 범용 ingress queue 서버다. RAGFlow는 core가 아니라 첫 번째 adapter이자 현재 LLM Wiki 저장소다. Summary/card 생성과 recall 판단은 queue 책임이 아니다.

01. System Map

Producer는 RAG target을 직접 write하지 않는다

workspace-ragflow-advisor의 현재 적용 대상은 Codex Stop hook이 남기는 TranscriptCaptureSpool과 기존 transcript worker다. 이 기존 경로를 Mac mini의 session-compactor로 재분류하고, Ubuntu의 memory-regeneration-runner는 RAGFlow를 읽어 derived docs를 다시 enqueue하는 batch producer로 둔다. Core server는 enqueue, JetStream publish, pull worker lifecycle, adapter 호출만 맡는다.

02. Runtime Boundary

RAGFlow compose project와 lifecycle을 분리한다

초기 구성은 별도 Docker Compose project로 시작한다. RAGFlow stack, Redis, DB, volume을 직접 조작하지 않고 host-published endpoint 또는 명시적으로 허용된 bridge만 사용한다.

Separate project

project: rag-ingress-queue로 시작한다. RAGFlow compose file은 수정하지 않는다.

Three services

nats-jetstream, ingress-api, ingress-worker. API와 worker는 같은 image, 다른 command로 실행할 수 있다.

No shared internals

RAGFlow 내부 Redis, DB, volume, parser state를 queue engine처럼 공유하지 않는다.

03. Workspace Session Compactor

완성된 session compactor의 write sink만 바꾼다

기존 hook 설치와 read plane은 그대로 둔다. 변경 지점은 현재 RAGFlow REST를 직접 호출하는 transcript ingest worker의 sink 구간이다. Mac mini에서는 source resolve, parse, redaction, packing까지만 수행하고 RAGFlow upload, parse request, status polling은 rag-ingress-queue worker delivery로 이동시킨다.

Existing surface 현재 역할 rag-ingress-queue 적용 후
UserPromptSubmit approved memory를 bounded additionalContext로 제공하는 read/recall path. 그대로 유지한다. Ingest queue에 넣지 않는다.
Stop + transcript-capture session_end boundary에서 locator/hash capture request 생성. 그대로 유지한다. Hook은 RAGFlow나 NATS를 직접 호출하지 않는다.
TranscriptCaptureSpool 0700 root와 0600 item으로 pending/processing/acked/quarantine 관리. producer-side durable inbox로 유지한다. publish ack 이후에만 spool ack.
TranscriptIngestWorker 현재는 parse, pack, RAGFlow upload/metadata/parse request/poll까지 수행. parse/redaction/pack은 session-compactor로 유지하고, 직접 RAGFlow upload/status poll 책임만 RAGFlowAdapter로 이동.
RAGFlow document status table session, turn, chunk, RAGFlow document ref, indexed status, authorization state 기록. canonical state table로 유지한다. queue ack와 authorization을 같은 상태로 취급하지 않는다. 
Immediate application path:
Codex Stop hook
  -> transcript-capture
  -> private TranscriptCaptureSpool
  -> session-compactor resolves local source on Mac mini
  -> parse + redaction.v2 + conversation chunk pack
  -> enqueue redacted RAGFlow-ready document
  -> POST /v1/ingest/enqueue
  -> JetStream publish ack
  -> local spool ack

Direct RAGFlow upload from the spool worker becomes
a legacy/pre-cutover path, not the steady-state path.

Batch regeneration path:
memory-regeneration-runner
  -> read RAGFlow transcript-memory
  -> generate derived docs
  -> POST /v1/ingest/enqueue
  -> RAGFlow derived datasets

04. Write Path

ingest write path는 하나만 허용한다

API accepted는 indexing 완료가 아니다. Local spool ack, JetStream publish ack, target indexed, RAGFlow 문서 상태표 authorization, recall/promote eligible은 서로 다른 상태로 추적한다.

Capture

Stop hook은 raw transcript 대신 locator_only capture request를 private spool에 쓴다.

Bridge

Session-compactor enqueue sink가 request를 검증한 뒤 Mac mini에서 source를 읽고 redacted RAG-ready document, contentHash, targetProfile job으로 변환한다.

Publish

ingress-api가 JetStream publish ack를 받은 뒤에만 local spool item을 ack할 수 있다.

Guard

Worker는 bounded batch를 pull하고 adapter pressure가 OPEN일 때만 target delivery를 수행한다.

Authorize

RAGFlow DONE 이후에도 문서 상태표 authorization pass 전에는 recall/promote 후보가 아니다.

05. Queue Contract

NATS JetStream에 queue 책임을 위임한다

Core server가 자체 queue engine을 만들지 않는다. persistence, explicit ack, redelivery, max deliver, ack wait은 JetStream stream과 consumer 설정으로 표현한다.

Surface Required shape Operator meaning Risk control
Session-compactor enqueue sink Existing TranscriptCaptureSpool item -> redacted RAG-ready document on rag.ingress.transcript 기존 hook path를 유지한 채 ingress queue producer로 연결한다. publish ack 전 local spool ack 금지. Mac-only locator, raw body, raw private path는 queue payload/response/log에 금지.
memory-regeneration-runner RAGFlow redacted corpus -> derived document on rag.ingress.document Batch regeneration도 direct RAGFlow write 없이 같은 ingress path를 사용한다. Runner는 queue consumer가 아니라 producer다. Summary/card 생성은 queue 책임이 아니다.
Stream RAG_INGRESS_QUEUE, WorkQueuePolicy, file storage Ack된 message는 work queue에서 제거된다. Queue persistence는 NATS가 맡고 app은 publish/delivery state만 관리한다.
Subjects rag.ingress.transcript, rag.ingress.document transcript와 document ingest를 같은 queue family 안에서 분리한다. subject naming으로 target kind 확장 여지를 둔다.
Consumer rag_target_delivery_worker, durable pull, explicit ack Worker가 target 상태에 맞춰 fetch rate를 조절한다. max_ack_pending, ack_wait, max_deliver를 bounded로 둔다.
Dead-letter max deliver 이후 redacted failure evidence 실패 payload 원문 대신 status class와 count를 보여준다. private path, token, raw target IDs는 노출하지 않는다.

06. Adapter Boundary

Core는 RAGFlow를 모른다

RAGFlowAdapter v1은 RAGFlow REST, parser backlog, document status를 generic pressure와 state로 바꾸는 첫 구현체다. 이후 target 교체는 adapter 추가로 끝나야 한다. 

public interface RagTargetAdapter {
    TargetPressure checkPressure(TargetProfile profile);
    DeliveryResult deliver(IngestJob job, TargetProfile profile);
    IndexingStatus getStatus(TargetDocumentRef ref, TargetProfile profile);
    ReconcileResult reconcile(TargetDocumentRef ref, DocumentStatusSnapshot statusState);
}

checkPressure

UNSTART, RUNNING, stale indexing, healthz, error rate를 OPEN, THROTTLED, CLOSED로 변환한다.

deliver

Core job을 target-specific upload와 parse request로 변환한다. targetProfile이 RAGFlow dataset role을 고르고, raw dataset_id와 document_id는 밖으로 내보내지 않는다.

getStatus

Target document status를 ACCEPTED, DELIVERED, INDEXING, INDEXED, FAILED로 해석한다.

reconcile

Target live 상태와 local document status snapshot이 어긋나면 redacted reconcile 후보를 만든다.

07. RAGFlow LLM Wiki Datasets

기억 본문은 RAGFlow dataset으로 나눈다

RAGFlow는 장기기억 본문과 검색 대상이 모이는 LLM Wiki다. 단일 dataset에 metadata type만으로 모든 기억 데이터를 넣지 않는다. targetProfile은 adapter가 선택할 RAGFlow dataset role을 나타낸다.

08. External Document Status Gate

indexed는 recall/promote eligible이 아니다

여기서 말하는 gate는 별도 장기기억 저장소가 아니라 RAGFlow document id/hash/state/provenance/authorization을 기록하는 external local 문서 상태표다. rag-ingress-queue는 이 상태표를 직접 소유하지 않고 redacted job/status snapshot을 제공한다. Queue ack와 document indexed는 같은 의미가 아니며, Target INDEXED 이후에도 authorization pass 전에는 transcript memory를 recall/promote에 사용하지 않는다.

captured

Private spool에 locator/hash request가 기록된 상태다. RAGFlow indexing 증거가 아니다.

ingested

Request가 JetStream에 publish되어 target delivery 후보가 된 상태다.

indexed

Target RAG가 indexing 완료 상태를 반환했다. 아직 authorization 의미는 없다.

authorized

Local 문서 상태표 authorization pass가 별도로 성공한 상태다.

recall/promote eligible

authorization 이후에만 eligible이다. pass 전 indexed memory promotion은 금지한다.

09. Operator Evidence

운영 출력은 redacted summary만 허용한다

Health, queue, target, document status 상태는 운영자가 판단할 수 있을 만큼 보여주되 token, raw dataset_id, raw document_id, private path, raw transcript body는 출력하지 않는다.

Spool

pending, processing, acked, quarantine count와 request parse status.

Compose

project status, service health, nats-jetstream, ingress-api, ingress-worker.

Queue

stream pending, consumer pending, ack pending, redelivered, dead-letter count.

Target

adapter name, target profile, dataset role, pressure state, recent delivery status class, reconcile candidate count.

Redaction

No tokens, raw IDs, raw private paths, raw transcript body, full raw RAG payload.
