배경

  • GraphitiNeo4jGraphMemoryAdapter 생성자는 이미 Graphiti instance를 받을 수 있지만, env 해석과 모델 조립이 adapter 파일에 묶여 있다.
  • Qdrant searchable mirror도 기존 OpenAI-compatible embedding/rerank endpoint 재사용을 전제로 한다.
  • rag_ingress 경로가 heavy llm_brain_core package를 끌어오지 않는 import guard가 중요하다.

질문-답변 흐름

Q1: 1차 성공 범위는 어디까지로 잡을까?

라이브 canary 포함으로 확정. code-only refactor와 fake/no-network test만으로 끝내지 않고, 구현 단계에서 별도 operator gate 이후 실제 Graphiti schema extraction canary까지 요구한다.
  • Phase 1/2 중에는 live 실행하지 않는다.
  • 구현 단계에서 current evidence, exact argv, bounded timeout, redaction, postcheck, rollback/abort 기준을 제시한다.
  • RetiredIndexBridge disable/delete, Qdrant cutover, native-memory authority 이전은 범위 밖이다.

Q2: Graphiti schema extraction canary는 어느 깊이까지 확인할까?

단일 synthetic episode로 확정. public-safe synthetic episode 하나로 Graphiti extraction, upsert, search를 확인한다.
  • 실제 projection slice나 live project memory를 태우지 않는다.
  • isolated brain_id / group scope를 사용한다.
  • raw transcript/source body, private path, secret-like 값, raw dataset/document id를 출력하지 않는다.
  • 성공 기준은 연결, 모델 기반 schema extraction, episode upsert, bounded search/readback 통과다.

Q3: 공용 model connector 책임은 어디가 소유할까?

낮은 shared layer로 확정. Graphiti/Qdrant adapter보다 낮은 계층이 소유하고, llm_brain_core와 rag_ingress는 서로를 import하지 않는다.
  • 구현 package 이름은 아직 확정하지 않는다.
  • 실제 위치와 public import path는 Phase 2 design에서 import guard와 compatibility를 검증하며 정한다.

Q4: GraphitiNeo4jConfig는 어떻게 다룰까?

장기 facade 유지로 확정. 기존 public surface는 유지하고 내부만 새 connector/builder 경로로 delegate할 수 있다.
  • GraphitiNeo4jConfig, GraphitiNeo4jGraphMemoryAdapter.from_config, GraphitiNeo4jGraphMemoryAdapter.from_env를 유지한다.
  • 기존 import path와 behavior는 깨지지 않아야 한다.
  • deprecation warning, 즉시 제거, 내부 전용화는 이번 범위 밖이다.

Q5: agent_knowledge root package rename은 이번 scope에 포함할까?

이번 scope에서는 현재 root 유지로 확정. 이번 요구사항은 model_connectors leaf package와 model connection boundary만 다룬다.
  • 기존 root import path인 agent_knowledge는 유지한다.
  • agent_knowledge rename은 별도 repo identity/API migration 요구사항으로 분리한다.

기능 요구사항

  • Graphiti adapter의 역할은 Graphiti instance를 통한 episode upsert/search로 제한한다.
  • 모델 연결 책임은 env 해석, non-secret model spec, policy, client 생성, fallback 구성으로 분리한다.
  • model connector shared layer는 llm_brain_core와 rag_ingress보다 낮은 위치에서 제공되어야 한다.
  • 이번 scope에서는 기존 agent_knowledge root import path를 바꾸지 않는다.
  • 기존 public compatibility와 env precedence를 유지한다.
  • GraphitiNeo4jConfig는 장기 compatibility facade로 유지한다.
  • secret-like 값은 repr, log, error, preview artifact에 노출하지 않는다.
  • Graphiti fallback, timeout, structured extraction normalization, graph activation behavior는 무회귀여야 한다.
  • 구현 검증은 fake/no-network test에 더해 public-safe synthetic episode 기반의 isolated live Graphiti schema extraction canary를 포함한다.
  • Qdrant import guard를 보존한다.
  • 정책 판정은 role/capability 단위로 한다.
