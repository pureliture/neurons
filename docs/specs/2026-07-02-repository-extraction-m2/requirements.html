Repository Extraction M2 Requirements

CurationService.approve를 첫 실제 repository extraction 대상으로 고정한다. production 검증은 read-only evidence로 제한한다.

확정 결정

  • 기존 승인 트랙 M2 Repository extraction을 구현한다.
  • 첫 caller migration은 CurationService.approve다.
  • MemoryCurationRepository port는 approve_candidate use-case만 소유한다.
  • supersede는 다음 multi-write 후보로 남기며 이번 milestone에서 transaction-safe 완료로 주장하지 않는다.
  • 이번 port는 active use-case seam이며 stable public import contract가 아니다.
  • ILedgerCoreDbAdapter, public UnitOfWork, REST/gRPC Ledger Core API는 범위 밖이다.
  • production 검증은 read-only evidence만 수행한다.

기능 요구사항

  • CurationService.approve는 repository port를 통해 write flow를 수행한다.
  • 기존 CurationService(ledger) construction은 유지한다.
  • memory card, evidence, candidate approval, profile fact write는 같은 transaction 안에서 실행한다.
  • Ledger._transaction()이 없으면 default repository는 fail-closed 한다.
  • repository metadata는 첫 실제 caller migration 상태와 public contract 미완료 상태를 함께 보여준다.
