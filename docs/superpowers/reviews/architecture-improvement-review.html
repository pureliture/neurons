별도 승인 또는 이후 단계 필요
D1
THROTTLED 느린 fetch 동작

MVP 스펙은 THROTTLED를 fail-closed로 정의합니다. 트리클 딜리버리는 별도 설정, 테스트, 배포 게이트가 필요합니다. 아키텍처 수정이 아니라 동작 변경입니다.

D2
Spring Security API 키 인증

MVP 범위 확장입니다. API 노출 범위가 먼저 확정되어야 합니다. 공개 엔드포인트 보호가 필요한지 검토 후 결정합니다.

D3
전체 어댑터 호출에 RetryPolicy

JetStream redelivery와 충돌합니다. 메시지 수준 재시도는 JetStream이 먼저 처리합니다. 어댑터 재시도는 멱등-안전 읽기에만 적용합니다.

제거된 / 중복 제안
예외 계층 전체 auto-wrap (GlobalResponseBodyAdvice) // /healthz, /status, 기존 API 테스트 계약을 깨뜨림
멱등성 재구현 // 이미 구현됨: idempotencyKey → Nats-Msg-Id
안전 제약 조건
라이브 RAGFlow 딜리버리: delivery-enabled=false, 별도 승인 전까지 금지
Docker/Compose 런타임 변경: compose.yaml 수정 금지
원격 호스트 변경: 금지
로컬 빌드 검증: JAVA_HOME + gradle test + postcheck.sh
코드 전용 변경: 런타임 부작용 없음
점진적: 광범위한 재작성 없음
